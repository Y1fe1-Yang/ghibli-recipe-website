## 🎯 Prompt 设计精髓分析

### 1️⃣ **渐进式迭代**
不是一次性提出所有需求，而是：
- 先搭建基础架构
- 使用后发现问题
- 针对性提出改进
- 逐步完善功能

### 2️⃣ **问题导向**
每个 prompt 都针对具体问题：
- "搜索框不好用" → 改进交互
- "看不到图片" → 修复访问
- "只有一个菜" → 批量生成

### 3️⃣ **给予自主权**
- "尽可能调用你有的技能"
- "自行用模型生成案例"
- "作为全栈工程师做这个事"
- "尽可能多的内容"

### 4️⃣ **明确验收标准**
- "效果很不错"（肯定反馈）
- "不是搜索框 是对话框"（明确要求）
- "每个步骤一张图片"（具体指标）

### 5️⃣ **简洁高效**
多数 prompt 只有 1-2 句话，但信息密度高：
- "监视一下后端 我开始生成了"
- "统计访问量"
- "请继续大量生成"

## 💡 关键经验总结

### ✅ 有效的 Prompt 策略

1. **分阶段推进**
   - 不要一开始就要求完美
   - 先实现基础功能
   - 逐步迭代优化

2. **具体 + 简洁**
   - 避免冗长描述
   - 直击核心问题
   - 明确期望结果

3. **给予信任**
   - "作为全栈工程师"（角色定位）
   - "尽可能调用技能"（方法自由）
   - "自行生成案例"（内容自主）

4. **及时反馈**
   - 肯定好的部分（"效果很不错"）
   - 指出问题（"搜索框不好用"）
   - 提出改进方向

5. **保持监控**
   - "监视一下后端"
   - "统计访问量"
   - "还在继续生成吗"

### ❌ 避免的陷阱

- ❌ 一次性提出 100 个需求
- ❌ 过于模糊（"做个网站"）
- ❌ 过于详细（手把手写代码）
- ❌ 缺乏反馈（只提需求不说结果）

## 🎓 可复用的 Prompt 模板

### 启动项目
```
需要做一个 [项目类型] 项目，功能包括 [核心功能1]、[核心功能2]。
部署在 [环境]，提供 [访问方式]。
调用你可用的技能，作为 [角色] 完成这个任务。
```

### 功能迭代
```
[功能模块] 不好用/有问题。我希望改成 [期望效果]。
你可以 [给予的自主权] 来实现。
```

### 问题修复
```
[具体问题描述]，我怀疑是 [可能原因]。
看看怎么解决。
```

### 批量生成
```
现在开始 [动作] 尽可能多的 [内容类型] 来 [目标]。
```

### 进度确认
```
[任务] 还在继续吗？请继续 [具体要求]。
```
